Introduction to Convolutional Neural Networks¶

One of the most common is the dense layer that connects all units in the layer directly to every unit in the previous layer. By stacking layers, the units in each subsequent layer can represent increasingly sophisticated aspects of the original input, such as learning the simplest Lego pieces at the first layer and then learning the linear combinations of Lego pieces to make up more complex pieces and then eventually learning the large complex structure that the data point(s) represent(s). Recall the error of each layer, which is minimized during training via the backpropagation algorithm.

A Convolutional Neural network CNN is a multiple-layer neural network that consists of convolutional layers, pooling layers, and fully connected layers as hidden layers. In a convolution layer, a filter is slid across the data, and feature maps are extracted through element-wise multiplication. Generally, a convolution layer has multiple filters, where one filter creates one feature map. The role of this layer is feature extraction.

The convolution is performed by multiplying the filter pixel-wise with the portion of the image and summing the result. The output is more positive when the portion of the image closely matches the filter and more negative when the portion of the image is the inverse of the filter. Let's move the filter across the entire image, from left to right and top to bottom, recording the convolutional output as we go. We obtain a new array that picks out a particular feature of the input depending on the values in the filter.

This is precisely what a convolutional layer is designed to do, but with multiple filters rather than just one. The example below shows two filters that highlight horizontal and vertical edges.

If we are working with color images, then each filter would have three channels rather than one (i.e., each having shape $3 \times 3 \times 3$) to match the three channels (red, green, blue) of the image.

Note that explainability (XAI) in machine learning refers to understanding and explaining why AI algorithms make their choices, making them transparent and interpretable to humans. A CNN is not more explainable than a regular NN because both are black-box classifiers.

PyTorch Convolutional Layer¶

Consider the Conv2D layer:

torch.nn.Conv2d(
  in_channels, out_channels, kernel_size, stride=1, padding=0,
  dilation=1, groups=1, bias=True, padding_mode='zeros',
  device=None, dtype=None)

Applies a 2D convolution over an input signal composed of several input planes (such as color planes).
The output value of the layer with input size $(N,C_{in},H,W)$ and output $(N,C_{out},H_{out},W_{out})$ can be described as:

$\text{out}(N_i,C_{out_j}) = \text{bias}(C_{out_j}) + \sum\limits_{k=0}^{C_{in}-1} \text{weight}(C_{out_j},k) \star \text{input}(N_i,k)$

where $\star$ is a cross-correlation operator, $N$ is the batch size, $C$ is the number of channels, $H$ is the height of the input planes (in pixels) and $W$ is the width (in pixels).

Conv2D Out Shape¶

To convert the Conv2D layer to a fully connected layer, the output shape is needed, and it is dependent on the input image size. Computing the shape of the convolutional layer output might be tricky:

$H_{out},W_{out}= \left\lfloor \dfrac{H_{in},W_{in} + 2 \times \text{padding} - \text{dilation}\times(\text{kernel}-1)-1}{\text{stride}} + 1 \right\rfloor$

Filters¶

Consider a convolutional filter of kernel_size=(H,W) $H \times W$ and a filter depth of $K$ (filters=K) where a portion of the input is mapped in 2-dimensions to a single neuron through the filter. If there is more than 1 channel of the image, such as R, G, and B, then these also add as $D$ many dimensions.

$z_{i,j} = \phi \bigg(b + \sum\limits_{l=0}^{k_H-1} \sum\limits_{m=0}^{k_W-1} \sum\limits_{n=0}^{D-1} w_{l,m,n} . x_{i+l,j+m,n} \bigg)$ and $w\in \mathbb{R}^{k_H \times k_W \times D}$ is the weight of the neuron.

Strides¶

The strides parameter is a 2-tuple of integers, specifying the step of the convolution along the x and y-axis of the input volume in pixels. Generally, it is left as the default (1, 1), occasionally increased to (2, 2) to help reduce the size of the output volume.

Padding¶

Padding is necessary to map the input $H \times W$ to the exact dimension of $H \times W$ at the output. $\frac{1}{2}$(kernel_size-1), many zeros are necessary on the left, right, top, and bottom boundaries of the input. Consequently, kernel_size has to be an odd number.

Subsampling Layers¶

A pooling $P$ or subsampling layer after a Conv2D layer reduces the spatial size of the representation to reduce the number of parameters (i.e., weights) and computation in the network. Max-pooling layers return only the maximum value at each depth of the pooled area, and average-pooling layers compute the average at each depth of the pooled area. The pooling layer operates on each feature map independently and generally halves each image dimension, e.g., $P_{2 \times 2}$ pooling.

Eventually, a flat layer is reached after convolutional layers (using Flatten()) and this layer is connected to the output layer with 10 categories to be classified in this example.

Batch Normalization¶

One common problem in training a deep neural network is ensuring the weights of the network remain within a reasonable range of values. If they start to become too large, then the gradients will grow (explode), and as errors are propagated backward through the network, the calculation of the gradient in the earlier layers can grow exponentially large, causing wild fluctuations in the weight values. If the loss function starts to return NaN, chances are that weights have grown large enough to cause overflow. This may not happen immediately at the start of the training, but sometimes the network and the loss function can suddenly start returning NaN.

One of the reasons for scaling input data into a neural network is to ensure a stable start to training over the first few iterations. Since the weights of the network are initially randomized, unscaled input could potentially create huge activation values that immediately lead to exploding gradients. For example, instead of passing pixel values from 0 to 255 into the input layer, we usually scale these values to between –1 and 1. Because the input is scaled, it's natural to expect the activations from all future layers to be relatively well-scaled as well. This may be true initially, but as the network trains and the weights move further away from their random initial values, this assumption can start to break down. This phenomenon is known as covariate shift.

Batch normalization drastically reduces this problem. A batch normalization layer calculates the mean and standard deviation of each input channel across the batch and normalizes by subtracting the mean and dividing by the standard deviation. Two learned parameters for each channel are the scale (gamma) and shift (beta). The output is simply the normalized input, scaled by gamma and shifted by beta.

Input: Batch $\text{B}=\{x_1, \dots, x_m\}$, learn parameters $\gamma$, $\beta$

Output: $y_i=\text{BN}_{\gamma,\beta}(x_i)$

Batch mean,
$\mu_{B} \leftarrow \frac{1}{m}\sum\limits{i=1}{m}x_i$

Batch variance,
$\sigma^2_{B} \leftarrow \frac{1}{m}\sum\limits{i=1}{m}(x_i - \mu_B)^2$

Normalize,
$\hat{x}_i \leftarrow \frac{x_i - \mu_B}{\sqrt{\sigma^2_{B}+\epsilon}}$

Scale and shift,
$y_i \leftarrow \gamma \hat{x}_i + \beta \equiv \text{BN}_{\gamma,\beta}(x_i)$

Place the batch normalization layers after dense or convolutional layers to normalize the output from those layers. In a way, those layers are connected by small sets of adjustable springs that ensure there aren't any huge shifts in their weights over time.

# Compute H, W for image size 28 - see the nn.Sequential below
def findConv2dOutShape(_H, _W, _conv, _pool=2):
    kernel_size, stride, padding, dilation = _conv.kernel_size, _conv.stride, _conv.padding, _conv.dilation
    H = np.floor((_H+2*padding[0]-dilation[0]*(kernel_size[0]-1)-1)/stride[0]+1)
    W = np.floor((_W+2*padding[1]-dilation[1]*(kernel_size[1]-1)-1)/stride[1]+1)
    if _pool:
        H, W = H/_pool, W/_pool
    return int(H), int(W)

def get_dataloader(_img_size, _bs, _ds, _path):
    train_ds = _ds(
        root=_path, download=False, train=True,
        transform=vtransforms.Compose([
            vtransforms.Resize(_img_size),
            vtransforms.ToTensor(),
            vtransforms.Normalize((0.5,), (0.5,))
        ]))

    # Use pin_memory=True to fix GPU memory
    train_dl = torch.utils.data.DataLoader(train_ds, batch_size=_bs, shuffle=True,
                                           # pin_memory=True,
                                           num_workers=4)

    test_ds = _ds(
        root=_path, download=False, train=False,
        transform=vtransforms.Compose([
            vtransforms.Resize(_img_size),
            vtransforms.ToTensor(),
            vtransforms.Normalize((0.5,), (0.5,))
        ]))

    test_dl = torch.utils.data.DataLoader(test_ds, batch_size=_bs, shuffle=True,
                                          # pin_memory=True,
                                          num_workers=4)

    return train_dl, test_dl

IMG_SIZE = 28
IMG_CHANNEL= 1  # color channel

MLP_HIDDEN= 16  # Hidden layer size

LAST_LINEAR_SIZE = 1024  # 1024 is arbitrary
N_CLASSES = 10  # output layer size

BATCH_SIZE = 5000

class PyTorchCNN(torch.nn.Module):
    def __init__(self, epochs=10, eta=0.001, batch_size=1000, seed=0):
        super().__init__()
        self.random = np.random.RandomState(seed)  # shuffle mini batches
        self.epochs = epochs  # number of iterations
        self.eta = eta  # learning rate
        self.batch_size = batch_size  # size of training batch - 1 would not work
        self.loss_func = torch.nn.CrossEntropyLoss()
        self.model = None

    def init_layers(self, _K):
        import torch.nn as nn
        self.model = nn.Sequential(
            # Conv -> (?, IMG_SIZE, IMG_SIZE, MLP_HIDDEN)
            # Pool -> (?, IMG_SIZE/2, IMG_SIZE/2, MLP_HIDDEN)
            nn.Conv2d(IMG_CHANNEL, MLP_HIDDEN, 5),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2),
            nn.BatchNorm2d(MLP_HIDDEN),

            nn.Conv2d(MLP_HIDDEN, MLP_HIDDEN*2, 5),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2),

            nn.Flatten(start_dim=1),
            
            # 4*4 is computed in the above cell - findConv2dOutShape
            nn.Linear(MLP_HIDDEN*2 * 4*4, LAST_LINEAR_SIZE),
            nn.BatchNorm1d(LAST_LINEAR_SIZE),
            
            nn.Linear(LAST_LINEAR_SIZE, _K),
        ).to(gpu)

    def predict(self, _X):
        _X = torch.FloatTensor(_X).to(gpu, non_blocking=True)
        assert self.model is not None
        with torch.no_grad():
            logits = self.model(_X)
        ypred = torch.argmax(logits, dim=1)
        return ypred.cpu().numpy()

    def fit(self, _train_dl, info=False):
        import sys
        self.init_layers(N_CLASSES)

        optimizer = torch.optim.Rprop(self.model.parameters(), lr=self.eta)

        for e in range(self.epochs):
            for data in _train_dl:
                X, y = data[0].to(gpu), data[1].to(gpu)

                optimizer.zero_grad()
                net_out = self.model(X)
                loss = self.loss_func(net_out, y)
                loss.backward()
                optimizer.step()
                
            if info:
                with torch.no_grad():
                    acc = accuracy_score(data[1].tolist(), np.argmax(self.model(X).cpu(), axis=1).tolist())
                sys.stderr.write(f"\r{e+1:02d}/{self.epochs:02d} | Loss: {loss:<6.2f} | Tr Acc: {acc*100:3.2f}%")
                sys.stderr.flush()

Audio Signal Classification¶

Recurrent neural networks can model sequences such as audio or text. However, audio or time-series data can be modeled by frequency domain representations as well.

The following Kaggle dataset has audio files in WAV format, which can be read by the torchaudio backend, soundfile library. The dataset is composed,

  • class 'cat' with 164 WAV files, 1323 sec of total audio
  • class 'dog' with 113 WAV files, 598 sec of total audio

The classical approach to modeling audio is based on Fourier transform and the spectrogram of audio or time-series signal. A spectrogram is revealed as an image with time as the x-axis and frequency information as the y-axis. While the details are skipped, the following Cells demonstrate the processing of one-dimensional digital audio sequences into two-dimensional spectrogram images. The hop-on time axis is set by FFT_HOP_N, and it corresponds to 100 msecs since the sampling frequency is 2KHz or 2000. As a result, 10 data points are extracted for 1 sec of audio. The number of frequency bins is selected as 20 with the parameter MFCC_M. As a result, a strip of 20 values per time slot (100 msecs) is generated during feature extraction.

Clearly, the variety of the duration of the signal makes it more difficult to handle audio data. Below, the histogram shows that most dog files are around 10 seconds (cats around 5 secs); thus cut it at 10 seconds, or top it off to 10 seconds by repeating the spectrogram. Ultimately, the spectrograms (20 by 100 images) of each cat or dog utterance are classified. Note that if each strip is considered as a single data point and classified (by assuming the entire set of strips are the same class), it may not work well due to background noise, etc. captured in the audio recording. Moreover, even this easy approach would yield around 80% classifier performance.

MFCC_N = 10*SAMPLING_FRQ//FFT_HOP_N  # cut it at 10 seconds or top up to 10 seconds by repeating circularly

# copy spectrogram vertical strips to extend the data point to size (20,100)
def topup(_X):
    __X = _X.copy()
    while _X.shape[1] < MFCC_N:
        _X = np.concatenate((_X, __X), axis=1)
    if _X.shape[1] > MFCC_N:
        _X = _X[:,:MFCC_N]
    return _X

def get_dataloader_audio(_X_tr, _X_ts, _y_tr, _y_ts, _bs):
    dataset_tr = torch.utils.data.TensorDataset(torch.tensor(_X_tr), torch.tensor(_y_tr))
    dataloader_tr = torch.utils.data.DataLoader(dataset_tr,
                                                sampler=torch.utils.data.RandomSampler(dataset_tr), batch_size=_bs)

    dataloader_ts = torch.utils.data.DataLoader(dataset_ts, batch_size=_bs)

MLP_HIDDEN= 8  # Hidden layer size

LAST_LINEAR_SIZE= 32
N_CLASSES= 2  # output layer size

BATCH_SIZE= 50

class PyTorchCNN2(PyTorchCNN):
    def __init__(self, epochs=10, eta=0.001):
        super().__init__(epochs=epochs, eta=eta)
        self.epochs = epochs  # number of iterations
        self.eta = eta  # learning rate
        self.loss_func = torch.nn.CrossEntropyLoss()
        self.model = None

    def init_layers(self, _K):
        import torch.nn as nn
        self.model = nn.Sequential(
            nn.Conv2d(1, MLP_HIDDEN, 5),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2),
            nn.BatchNorm2d(MLP_HIDDEN),

            nn.Conv2d(MLP_HIDDEN, MLP_HIDDEN*2, 5),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2),

            nn.Flatten(start_dim=1),
            
            # 2*22 is computed in the above cell - findConv2dOutShape
            nn.Linear(MLP_HIDDEN*2 * 2*22, LAST_LINEAR_SIZE),
            nn.BatchNorm1d(LAST_LINEAR_SIZE),
            
            nn.Linear(LAST_LINEAR_SIZE, _K),
            nn.Tanh()
        ).to(gpu)

# create a dataloader for every fold - inefficient
def kfold_eval(_clf, _X, _y):
    from sklearn.model_selection import StratifiedKFold
    accuracies = []
    kf = StratifiedKFold(n_splits=10, shuffle=True, random_state=0)
    for train_ix, test_ix in kf.split(_X, _y):
        dla_tr, dla_ts = get_dataloader_audio(_X[train_ix], _X[test_ix], _y[train_ix], _y[test_ix], BATCH_SIZE)
        _clf.fit(dla_tr)
        accuracies += [predict_dla(_clf, dla_ts)]
    return np.array(accuracies)
