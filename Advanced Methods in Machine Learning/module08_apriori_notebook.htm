Apriori Analysis¶

Introduction¶

Apriori analysis and apriori algorithm for frequent itemset mining and association rule learning is used to find patterns in transactional datasets. An example is from online retailers where the page suggests the prospective buyer with the comment: "Who buys this also buys that". Today, online retailers must show customers the most relevant page since they carry millions of items for sale. This relevancy can be computed with apriori analysis. Another example is brick-and-mortar store product placement, such as placing related items close to each other so that the buyer does not have to walk much or the buyer buys it when he/she sees a related item. Famous beer-diaper story - Originated in 1992, according to an industry consultant, Teradata, when analyzing a retailer's database, discovered that there is a good chance when a customer buys diaper, the customer also buys beer. Of course, this discovered pattern was subtle but resulted in a statistically probable frequent itemset after the apriori analysis of the transactional dataset.

Association Rule Learning¶

A method to determine relations among variables in large databases is as follows:

Let $I=\{i_1, i_2, \dots, i_m\}$ be a set of $m$ features, representing distinct items (i.e., products).

Let $D=\{t_1, t_2, \dots, t_n\}$ be the set of $n$ transactions stored in a database $D$. Every $t_i$ is a distinct event and consists of a subset of items from $I$.

A rule is an implication $X \rightarrow Y$ where $X$ and $Y$ are itemsets and subsets of $I$, and they have no element in common, i.e. $X \cap Y = \emptyset$

$X$ is called antecedent and $Y$ is called consequent of the rule.

An example could be {statistics book, algorithms book, programming book} $\rightarrow$ {machine learning book}

Clearly, extracting such relations from the datasets is a tedious task and naively has exponential time complexity. Given $m$ items, there are $2^m$ possible candidate itemsets.

Support¶

The support of an itemset $X$, $\text{supp}(X)$ is the ratio of the number of transactions, which includes all the items of $X$ to the total number of transactions. Naturally, it shows the popularity of the itemset, i.e. {CD recorder, CD blank media}

$\operatorname{supp}(X)=\dfrac{\text{number of} X \subseteq \text{transactions}}{\text{total number of transactions}}=\dfrac{|\{t \in D; X \subseteq t\}|}{|D|}$ .................... $(Eq.1)$

For example, online retailers show item rankings to guide buyers. Computing all itemsets with one distinct product would give that ranking.

Confidence¶

Confidence (also corresponds to conditional probability) signifies the likelihood of item $Y$ being purchased when item $X$ is purchased.

$\operatorname{conf}(X \rightarrow Y)=\dfrac{\operatorname{supp}(X \cup Y)}{\operatorname{supp}(X)}$ .................... $(Eq.2)$

Lift¶

Lift shows the likelihood of itemset $Y$ when $X$ is purchased while normalizing with respect to the popularity of $Y$.

$\operatorname{lift}(X \rightarrow Y)=\dfrac{\operatorname{supp}(X \cup Y)}{\operatorname{supp}(X)\operatorname{supp}(Y)}$ .................... $(Eq.3)$

Notes on the Application and Algorithm¶

Considering transactional datasets where a buyer might have more than one transaction, the purchases of the same buyer can be combined, i.e., weekly or monthly, to make into itemsets, assuming the buyer purchases in a month have relevancy between the items purchased for a buyer.
Apriori algorithms such as Max-Miner, FP-growth use the following facts (apriori principle below):
- All subsets of a frequent itemset must be frequent
- Any infrequent itemset, all its supersets must be infrequent

Thus, applying a support threshold to prune the itemsets that have lower support decreases the amount of data to be processed.

Generating frequent k-itemset corresponds to finding all itemsets that have k items in them with a support higher than a given minsupport threshold. Clearly, in order to generate efficiently the k-itemset, we compute (k-1)-itemset.

Apriori Principle¶

Anti-monotone property of support: Support of an itemset never exceeds the support of its subsets.

$\forall X, Y : (X \subseteq Y) \implies \operatorname{supp}(X) \ge \operatorname{supp}(Y)$ .................... $(Eq.4)$

Let's examine and run the code below on the store data file and process frequent itemsets that exist in the dataset. The dataset contains items purchased from a store, such as salad, turkey, tomatoes, etc. Each row is a list of items that were purchased in the same transaction by the customer.

The following cells will use a conversion, called denormalization, on the dataset list of transactions to another system, where each item presence is represented by a True or False, i.e., 0 or 1. Note that the Weka framework also uses this representation.

Prospect Improvement¶

As can be seen by the enormous number of possible itemsets, it becomes infeasible to directly compute the support for k-itemsets, as k increases. Thus, a simple check has to be added to the support function, such that when a support of '$k<$' subset itemset is $0$, the support for the k-itemset is returned as $0$. This corresponds to the primitive application of $Eq.4$. This improvement still will not be enough to speed up computations, thus algorithms such as Max-Miner or FP-Growth are necessary.

# Based on equations
def support(_itemset:list)->int:
    global Transactions, N, M
    cnt = 0
    for transaction_i in range(N):  # all transactions
        bMatch = True
        for item_ in _itemset:
            if not Transactions[transaction_i][item_]:
                bMatch = False
                break
        if bMatch:
            cnt += 1
    return cnt

def confidence(_it1:list, _it2:list)->float:
    return support([_it1, _it2])/support([_it1])

Rules¶

A rule can be computed through the Confidence in $Eq.2$, which shows how often the rule is true in the dataset.

The confidence value of a rule,
$X \rightarrow Y$
is the proportion of the transactions that contains $X$ which also contains $Y$.

Following cells demonstrate rules extracted from 2-itemsets and 3-itemsets.

The discovered first rule says that if the item 'asparagus' is purchased, then there is a 100% chance that ('shrimp', 'burgers') are also purchased. Since support or the probability of an itemset in the dataset drives most of this derivation, Apriori analysis works best on huge amounts of transactions, otherwise discovered rules might be just misleading. For example, the support of 'asparagus' is very low, i.e., 1 count, and whenever it shows up, the rest of the items ('shrimp', 'burgers') will be purchased since it is the only transaction we have with 'asparagus'. Thus, the rule is correct but useless (e.g., it can be considered as an anomaly).

As the above cells show, fast algorithms using the Apriori principle are mandatory to work with large datasets.

def save_file(_fn):
    global Transactions, Items_names, N, M
    with open(_fn, 'w') as fout:
        writer = csv.writer(fout, delimiter=',', quoting=csv.QUOTE_ALL, quotechar="'", lineterminator='\n')
        writer.writerow([Items_names[i] for i in range(M)])
        for i in range(N):
            writer.writerow(list(map(lambda x: '' if x == False else 'True',  Transactions[i])))
