Reinforcement Learning¶

Introduction¶

Reinforcement Learning (RL) is a form of learning from experience through reward and penalty.
The approach is used in Artificial Intelligence, Control Theory, Psychology, Neuroscience, and Artificial Neural Networks. What makes this different from supervised learning?

  • Supervised Learning: The goal is to minimize an error where the error is the difference between target output and actual output. You can compute the error because you have labeled observations (training data) that you used to build your model.
  • Reinforcement Learning: The goal is to maximize the reward given by training evaluation (rewards and penalties). The model learns by confronting examples that don't have to exist beforehand (e.g., competing against random players in games).

Reinforcement learning also involves policy learning (how to map situations to actions), goal-oriented learning, and learning from interaction (i.e., games).

Features of RL¶

  • Trial-and-Error search
  • Learner decides which actions to take
  • The need to balance exploration (of unfamiliar strategies) and exploitation (of tried-and-true actions)
  • The whole problem of a goal-directed agent interacting with an uncertain environment
  • Might yield a delayed reward (forgo short-term gains for greater long-term gains)

Agent Based Learning¶

Consider an environment and 2 agents:

The agent continuously learns and plans. The objective is to affect the environment, which can be stochastic and uncertain. The learner employs a reward/penalty strategy to enable learning. The strength of this strategy also determines the speed of the learning. The learner retains the experience that yields gain.

Elements of RL¶

  • Policy: what to do
  • Reward: what is good
  • Value: what is good because it predicts reward
  • Model: what follows what

$\text{Policy} \sqsubset \text{Reward} \sqsubset \text{Value} \sqsubset \text{Model}$

Evaluative Feedback¶

Evaluating actions $\longleftrightarrow$ instructing by giving correct actions

Pure evaluative feedback depends completely on the action taken
Pure instructive feedback depends not at all on the action taken

Supervised learning is instructive; optimization is evaluative.

  • Associative: Inputs mapped to Outputs; learn the best output for each input
  • Non-associative: learn (find) one best Output

n-armed bandit - Non-associative, evaluative feedback

n-Armed Bandit Problem¶

A slot machine is nicknamed a "one-armed bandit." A gambler with a set amount of money and is trying to allocate that money between machines (which machines to play? in what order? Will he play some machine multiple times?) is said to have a "multi-armed bandit problem."

In general, the multi-armed bandit problem involves a fixed, limited set of resources that must be allocated between competing or alternative choices to maximize their expected gain. These gains cannot be immediately computed, and as the learning proceeds, gains (or losses) will materialize. The problem represents the exploration versus exploitation trade-off.

Q-value: When an agent takes an action $a$ in state $s$, at time $t$, the predicted future rewards is defined as $Q(s_t,a_t)$

(Note: The term Q is not from the Star Trek entity Q, but stands for the Quality of the action while learning.)

Choose repeatedly from one of $n$ actions where each choice is called a play. After each play $a_t$, you get a reward $r_t$, where $E(r_t|a_t)=Q^*(a_t)$.

The distribution of $r_t$ depends only on $a_t$. The objective is to maximize reward in the long term, e.g., over 1000 plays.

To solve the n-armed bandit problem, we have to explore a variety of actions and then exploit the best of them.

Exploration and Exploitation¶

Greedy action at t is the following:

$a^*_t = \underset{a}{\texttt{argmax}}Q_t(a)$ .................... $(Eq.1)$

$a_t = a^*_t \implies \text{exploitation}$
$a_t \ne a^*_t \implies \text{exploration}$

Reinforcement learning requires a balance between exploration and exploitation.

Q-Learning¶

Based on the Bellman equation, the expected value of states and actions are stored in a table called the Q-table $Q(s,a)$. The Q-table indicates to the agent (learner) the expected reward for taking action $a$ in state $s$. The values of the Q-table are updated using the equation,

$Q(s,a) \longleftarrow Q(s,a)+ \alpha \ \big(R(s)+ \gamma \ \underset{a'}{\texttt{max}}Q(s',a')-Q(s,a) \big)$ .................... $(Eq.2)$

We can factor out a mutiple of $(1 - \alpha)$ from coefficients of the $Q(s,a)$ term and rearrange that equation:

$Q(s,a) \longleftarrow (1-\alpha)\underbrace{Q(s,a)}_\text{old value} + \underbrace{\alpha}_\text{learning rate} \big(\underbrace{R(s)}_\text{reward} + \underbrace{\gamma}_\text{discount factor} \underbrace{\underset{a'}{\texttt{max}}Q(s',a')}_\text{estimate of optimal future value} \big)$ .................... $(Eq.3)$

The Q values are updated with respect to the action that maximizes the expected reward in the successor state with the equation, $\underset{a'}{\texttt{max}}Q(s',a')$

$0<\alpha<1$ is the learning rate parameter; closer to 1 reward recent actions more
$0<\gamma<1$ is the discount factor, closer to 1 rewards future actions more
$R(s)$ is the reward for state $s$

The parameters of the update formula must be tuned for the agent to converge to the optimal policy as fast as possible.

Learning the Nim Game¶

Nim is a game of strategy between two players in which players remove items from three piles on each turn. For every turn, the player must remove at least one item from exactly one of the piles. There are two different versions of the game goal:
(A.) the player who clears the last pile wins
(B.) the player who clears the last pile (by removing 1 $\le$ items) loses

For the following Nim game learning example, for simplicity, in this notebook, we will consider the goal of the game to be to clear the last pile (game (A.) above).

We have three computer-automated players:

  1. Random player, who moves randomly
  2. Guru player, who moves according to the Nim solution
  3. Q-learner

The following is a simple implementation of the game and the learning strategy. st[i] is the state for a pile i=0,1,2. The number of piles is hard-coded to 3, and the number of max items per pile is ITEMS_MX =10 in the following example. Note that pile symmetry is not handled.

Note that updates to the qtable should be through the strategy, but not manually as this may break the weights and weight updates the qtable that is being accumulated. A consistent weighing across the entire qtable is important.

import numpy as np
from random import randint, choice

# The number of piles is 3


# max number of items per pile
ITEMS_MX = 10

# Initialize starting position
def init_game()->list:
    return [randint(1,ITEMS_MX), randint(1,ITEMS_MX), randint(1,ITEMS_MX)]

# Based on X-oring the item counts in piles - mathematical solution
def nim_guru(_st:list)->(int,int):
    xored = _st[0] ^ _st[1] ^ _st[2]
    if xored == 0:
        return nim_random(_st)
    for pile in range(3):
        s = _st[pile] ^ xored
        if s <= _st[pile]:
            return _st[pile]-s, pile

# Random Nim player
def nim_random(_st:list)->(int,int):
    pile = choice([i for i in range(3) if _st[i]>0])  # find the non-empty piles
    return randint(1, _st[pile]), pile  # random move

In the following cell the Q-learner is presented. The function makes a move according to the learned Q-table. The Q-table format is a matrix of type float and has dimensions (number_of_states, possible_moves) which covers all possible states and actions there can be for the Nim game.

The states are simply a list of the number of items in the 1st pile, the number of items in the 2nd pile, and the number of items in the 3rd pile.
e.g., st = [10,10,10] can be a starting state

The move or action is the tuple of the number of items to remove and the pile ID.
e.g., a = (2,0) can be an action

For ease of programming and assuring the Q-table does not contain illegal moves (e.g., move (0,0) - not removing an item from a pile), the following table indexing is used:
index = pile*ITEMS_MX + move-1 $\longleftrightarrow$ move, pile = index%ITEMS_MX+1, index//ITEMS_MX

Note that indexing is 0-based.

The first index part is the pile, i.e., 0, 1, 2. The second index part uses a numbering system, simply based on the number of max items per pile, i.e. using max items of 10. Examples,
index 2 $\longrightarrow$ move 3, pile 0
index 11 $\longrightarrow$ move 2, pile 1
move 10, pile 2 $\longrightarrow$ index 29

Note: In RL, the model developer designs the states and the actions (i.e. game move).

def nim_qlearner(_st:list)->(int,int):
    global qtable
    # pick the best rewarding move, equation 1
    a = np.argmax(qtable[_st[0], _st[1], _st[2]])  # exploitation
    # index is based on move, pile
    move, pile = a%ITEMS_MX+1, a//ITEMS_MX
    # check if qtable has generated a random but game illegal move - we have not explored there yet
    if move <= 0 or _st[pile] < move:
        move, pile = nim_random(_st)  # exploration
    return move, pile  # action

We will use the engines nim_guru, nim_random and nim_qlearner to play games, as below.

Engines = {'Random':nim_random, 'Guru':nim_guru, 'Qlearner':nim_qlearner}

def game(_a:str, _b:str):
    state, side = init_game(), 'A'
    while True:
        engine = Engines[_a] if side == 'A' else Engines[_b]
        move, pile = engine(state)
        # print(state, move, pile)  # debug purposes
        state[pile] -= move
        if state == [0, 0, 0]:  # game ends
            return side  # winning side
        side = 'B' if side == 'A' else 'A'  # switch sides

def play_games(_n:int, _a:str, _b:str)->(int,int):
    from collections import defaultdict
    wins = defaultdict(int)
    for _ in range(_n):
        wins[game(_a, _b)] += 1
    # info
    print(f"{_n} games, {_a:>8s}{wins['A']:5d}  {_b:>8s}{wins['B']:5d}")
    return wins['A'], wins['B']

Now let's see how the Q-learner actually learns by the Q-learning algorithm.

Alpha $\alpha$ is set to 1 since the Nim game is deterministic and every winning state must be learned.
For stochastic environments, where events happen randomly or cannot be predicted, alpha is set to less than 1.

qtable, Alpha, Gamma, Reward = None, 1.0, 0.8, 100.0

# learn from _n games, randomly played to explore the possible states
def nim_qlearn(_n:int):
    global qtable
    # based on max items per pile
    qtable = np.zeros((ITEMS_MX+1, ITEMS_MX+1, ITEMS_MX+1, ITEMS_MX*3), dtype=np.float32)
    # play _n games
    for _ in range(_n):
        # first state is starting position
        st1 = init_game()
        while True:  # while game not finished
            # make a random move - exploration
            move, pile = nim_random(st1)
            st2 = list(st1)
            # make the move
            st2[pile] -= move  # --> last move I made
            if st2 == [0, 0, 0]:  # game ends
                qtable_update(Reward, st1, move, pile, 0)  # I won
                break  # new game

            qtable_update(0, st1, move, pile, np.max(qtable[st2[0], st2[1], st2[2]]))
            
            # Switch sides for play and learning
            st1 = st2

# Equation 3 - update the qtable
def qtable_update(r:float, _st1:list, move:int, pile:int, q_future_best:float):
    a = pile*ITEMS_MX+move-1
    qtable[_st1[0], _st1[1], _st1[2], a] = Alpha * (r + Gamma * q_future_best)

# Function to print the entire set of states
def qtable_log(_fn:str):
    with open(_fn, 'w') as fout:
        s = 'state'
        for a in range(ITEMS_MX*3):
            move, pile = a%ITEMS_MX+1, a//ITEMS_MX
            s += ',%02d_%01d' % (move,pile)
        print(s, file=fout)
        for i, j, k in [(i,j,k) for i in range(ITEMS_MX+1) for j in range(ITEMS_MX+1) for k in range(ITEMS_MX+1)]:
            s = '%02d_%02d_%02d' % (i,j,k)
            for a in range(ITEMS_MX*3):
                r = qtable[i, j, k, a]
                s += ',%.1f' % r
            print(s, file=fout)
